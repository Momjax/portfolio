Réalisations RP1 - Classcord

RP1 - Classcord

Application de messagerie instantanée en Java inspirée de Discord

Terminé Java Swing TCP/JSON

1. Contexte

Situation

Ce projet a été réalisé dans le cadre du BTS SIO SLAM lors d'une semaine intensive de développement Java. L'objectif était de créer une application de messagerie fonctionnelle démontrant nos compétences en programmation réseau et interface graphique.

Organisation

Projet développé en binôme avec un camarade SISR. J'ai principalement travaillé sur le développement client et l'interface graphique, en collaboration sur le protocole de communication.

Problématique

Comment concevoir une application de messagerie temps réel permettant à plusieurs utilisateurs de communiquer via une architecture client-serveur fiable ?

2. Besoin & Objectifs

Objectifs fonctionnels

  • Connexion multi-utilisateurs simultanée
  • Envoi/réception de messages en temps réel
  • Messages publics et privés
  • Liste des utilisateurs connectés avec statuts
  • Changement de statut (disponible, absent, invisible)

Contraintes techniques

  • Java + Swing obligatoire
  • Sockets TCP pour le réseau
  • Format JSON pour les échanges
  • Architecture MVC

3. Solution Proposée

Architecture client-serveur

L'application Classcord utilise une architecture classique :

  • Serveur : Gère les connexions, route les messages, maintient l'état
  • Client : Interface Swing pour l'interaction utilisateur
  • Protocole : Messages JSON via sockets TCP
┌─────────────┐                  ┌─────────────┐
│   Client 1  │ ←── TCP/JSON ──→ │             │
└─────────────┘                  │   SERVEUR   │
┌─────────────┐ ←── TCP/JSON ──→ │    Java     │
│   Client N  │                  │             │
└─────────────┘                  └─────────────┘

Structure JSON des messages

// Connexion
{ "type": "LOGIN", "username": "Mohamed", "status": "AVAILABLE" }

// Message public
{ "type": "MESSAGE", "from": "Mohamed", "to": "ALL", "content": "Bonjour !" }

// Message privé
{ "type": "PRIVATE_MESSAGE", "from": "Mohamed", "to": "Alice", "content": "..." }

4. Réalisation

Ma contribution

  • Interface graphique Swing (fenêtre chat, liste utilisateurs, connexion)
  • Gestion des événements (listeners, actions utilisateur)
  • Client socket et parsing JSON
  • Affichage temps réel des messages et utilisateurs

Extrait de code - Envoi de message

public void sendMessage(String content, String recipient) {
    JSONObject msg = new JSONObject();
    msg.put("type", recipient.equals("ALL") ? 
            "MESSAGE" : "PRIVATE_MESSAGE");
    msg.put("from", this.username);
    msg.put("to", recipient);
    msg.put("content", content);
    outputStream.writeUTF(msg.toString());
}

6. Difficultés & Solutions

Problème 1 : Synchronisation des threads

L'interface se figeait lors de la réception de messages.

Solution :

Utilisation de SwingUtilities.invokeLater() pour les mises à jour UI depuis le thread réseau.

Problème 2 : Parsing JSON instable

Erreurs lors de messages mal formatés.

Solution :

Try-catch systématique et validation du JSON avant traitement.
