RP2 - NutriFit

Application web de suivi nutritionnel et bien-être

Terminé HTML5 CSS3 JavaScript

1. Contexte

Situation

Projet pédagogique réalisé dans le cadre du BTS SIO SLAM. L'objectif était de créer une application web interactive permettant aux utilisateurs de suivre leur alimentation et d'optimiser leur programme nutritionnel.

Organisation

Développé en binôme avec un camarade SISR sur une période de 6 semaines, avec des sprints hebdomadaires pour valider les fonctionnalités.

Problématique

Comment concevoir une application web intuitive permettant aux utilisateurs de calculer leurs besoins caloriques, rechercher des aliments et suivre leur progression ?

2. Besoin & Objectifs

Objectifs fonctionnels

  • Calculer le BMR (métabolisme de base) selon les données utilisateur
  • Rechercher des aliments avec leurs valeurs nutritionnelles
  • Générer un profil personnalisé avec recommandations
  • Afficher des graphiques de progression
  • Exporter le profil en PDF

Contraintes techniques

  • Technologies web front-end uniquement (HTML/CSS/JS)
  • Design responsive (mobile-first)
  • Intégration d'API pour les données nutritionnelles
  • Interface utilisateur intuitive

3. Solution Proposée

Architecture de l'application

Application web monopage (SPA) avec :

  • HTML5 : Structure sémantique de la page
  • CSS3 + Bootstrap : Design responsive et moderne
  • JavaScript : Logique métier et interactions
  • API externe : Données nutritionnelles des aliments

Extrait de code - Calcul BMR


function calculateBMR(weight, height, age, gender) {

    let bmr;

    if (gender === 'male') {

        bmr = 88.362 + (13.397 * weight) 

            + (4.799 * height) - (5.677 * age);

    } else {

        bmr = 447.593 + (9.247 * weight) 

            + (3.098 * height) - (4.330 * age);

    }

    return Math.round(bmr);

}

6. Difficultés & Solutions

Problème 1 : CORS avec l'API

Erreurs CORS lors des appels à l'API nutritionnelle.

Solution :

Utilisation d'une API avec support CORS natif et headers appropriés.

Problème 2 : Responsive des graphiques

Les graphiques Chart.js ne s'adaptaient pas correctement sur mobile.

Solution :

Configuration de l'option responsive: true et maintainAspectRatio: false.
