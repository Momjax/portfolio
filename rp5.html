RP5 - MediaStock

Application web de gestion de prêt de matériel audiovisuel

Terminé HTML5 CSS3 JavaScript

1. Contexte

Situation

Ce projet a été réalisé dans le cadre de ma 2ème année de BTS SIO SLAM. L'objectif était de créer une application web permettant de gérer le prêt et le retour de matériel audiovisuel (caméras, micros, éclairages, etc.) dans un établissement scolaire ou une structure culturelle.

Organisation

Projet développé en binôme avec un autre étudiant SLAM sur une période de 6 semaines. Nous avons travaillé en méthode agile avec des sprints hebdomadaires.

Problématique

Comment créer une application web intuitive permettant de suivre en temps réel les prêts de matériel, d'éviter les conflits de réservation et de maintenir un historique complet des emprunts ?

2. Besoin & Objectifs

Objectifs fonctionnels

  • Inventaire du matériel disponible
  • Système de création de prêts
  • Suivi de l'état du matériel (disponible, emprunté, en maintenance)
  • Historique des prêts et retours
  • Interface utilisateur simple et rapide
  • Recherche et filtrage du matériel

Contraintes techniques

  • Application web front-end uniquement
  • Stockage local (localStorage)
  • Design responsive et moderne
  • Interface intuitive sans formation

3. Solution Proposée

Architecture de l'application

Application web monopage (SPA) avec :

  • HTML5 : Structure sémantique de la page
  • CSS3 : Design moderne avec flexbox/grid
  • JavaScript : Logique métier et gestion des données
  • localStorage : Persistance des données côté client

Extrait de code - Création d'un prêt

function creerPret(materielId, emprunteur, dateDebut, dateFin) {
    const pret = {
        id: `PRET${Date.now()}`,
        materielId: materielId,
        emprunteur: emprunteur,
        dateDebut: dateDebut,
        dateFin: dateFin,
        dateRetour: null,
        statut: 'en cours'
    };
    
    // Sauvegarder dans localStorage
    let prets = JSON.parse(localStorage.getItem('prets')) || [];
    prets.push(pret);
    localStorage.setItem('prets', JSON.stringify(prets));
    
    // Mettre à jour le statut du matériel
    updateMaterielStatut(materielId, 'emprunté');
}

6. Difficultés & Solutions

Problème 1 : Gestion du localStorage

Difficulté à synchroniser les données entre les différentes vues.

Solution :

Création d'un module centralisé de gestion des données avec événements personnalisés pour la mise à jour.

Problème 2 : Validation des dates

Empêcher les dates incohérentes (fin avant début).

Solution :

Validation JavaScript avec comparaison de dates et messages d'erreur explicites.

8. Gestion de Projet

Pour ce projet, nous avons utilisé une approche Agile combinant Scrum, MoSCoW et Kanban :

Méthodologie Scrum

Organisation en sprints de 1 semaine pour un développement itératif rapide :

  • Sprint Planning : Définition des tâches chaque lundi
  • Daily Standup : Point quotidien de 15 minutes
  • Sprint Review : Démonstration des fonctionnalités
  • Sprint Retrospective : Analyse d'amélioration
