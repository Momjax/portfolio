# Justification des Compétences du Tableau de Synthèse E4

Ce document justifie, pour chaque réalisation professionnelle inscrite dans le tableau E4, pourquoi les compétences ont été cochées. Basé sur le fichier officiel `TABLEU DE SYNTHESE FINAL MOMO.pdf`.

> **Règle** : On coche une compétence si le projet mobilise **au moins un** des points qui la composent.
> **B1.3** (Développer la présence en ligne) n'est pas cochée pour les projets de formation — uniquement pour les stages.

---

## RP1 – Classcord (16/09/24 au 27/09/24)
*Compétences cochées : **B1.1, B1.2, B1.4, B1.5, B1.6***

- **B1.1 – Gérer le patrimoine informatique** : Gestion du code source sur Git, configuration de l'environnement (JDK, IDE), identification des ressources serveur.
- **B1.2 – Répondre aux incidents/demandes** : Réponse à un besoin fonctionnel (messagerie réseau local), gestion des erreurs de connexion via les exceptions Java.
- **B1.4 – Travailler en mode projet** : Conception de l'architecture client/serveur, découpage des tâches, itérations progressives (terminal → Swing).
- **B1.5 – Mettre à disposition un service** : Déploiement d'un serveur TCP actif, livraison d'un JAR exécutable permettant la communication en temps réel.
- **B1.6 – Développement professionnel** : Apprentissage autonome des Sockets Java, des flux I/O et du multi-threading.
