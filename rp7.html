Réalisations RP7 - RoomBooking

RP7 - RoomBooking

Application web de réservation de salles — Symfony 7, Doctrine ORM, Docker, déploiement en production

Terminé Symfony 7 Twig Doctrine Docker MySQL

1. Contexte

Situation

Ce projet a été réalisé dans le cadre de ma 2ème année de BTS SIO SLAM à l'École IRIS Nice (MediaSchool). L'objectif était de développer une application web complète de gestion de réservation de salles, en suivant un cahier des charges découpé en 20 jalons.

Organisation

Projet développé individuellement sur plusieurs semaines, avec une rigueur de gestion de projet : conventions de commits Git, guide de déploiement, tests unitaires avec PHPUnit.

Problématique

Comment concevoir une application web robuste permettant aux utilisateurs de réserver des salles de cours/réunion, avec gestion des rôles (administrateur / utilisateur), déploiement sur une VM Linux de l'école et sécurisation de l'accès ?

2. Besoin & Objectifs

Objectifs fonctionnels

  • Authentification sécurisée (login / logout)
  • Gestion des rôles : Administrateur et Utilisateur
  • Réservation de salles avec créneaux horaires
  • Panneau d'administration pour gérer salles et utilisateurs
  • Tests unitaires automatisés (PHPUnit)
  • Déploiement en production sur VM Linux de l'école

Contraintes techniques

  • Framework Symfony 7 obligatoire
  • ORM Doctrine pour la base de données
  • Containerisation avec Docker
  • Déploiement SSH sur VM Linux (Apache / MariaDB)
  • Conventions de commits à respecter

6. Difficultés & Solutions

Problème 1 : Configuration Docker en production

Les variables d'environnement différaient entre le dev et la VM de production.

Solution :

Utilisation de fichiers .env, .env.dev et .env.local séparés, avec guide de déploiement documenté.

Problème 2 : Gestion des migrations Doctrine

Conflits lors des modifications de schéma en cours de développement.

Solution :

Utilisation systématique de doctrine:migrations:migrate et suivi des versions avec Git.
