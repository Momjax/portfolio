RP4 - E-Consulat

Plateforme web de gestion consulaire avec système de prise de rendez-vous

Terminé HTML5 CSS3 JavaScript PHP MySQL

1. Contexte

Situation

Ce projet a été réalisé dans le cadre de ma 2ème année de BTS SIO SLAM. L'objectif était de créer une plateforme web complète pour faciliter les démarches administratives des ressortissants algériens en France, en digitalisant les services consulaires.

Organisation

Projet développé en autonomie sur une période de 8 semaines, avec un cahier des charges définissant les besoins fonctionnels et techniques.

Problématique

Comment concevoir une plateforme web sécurisée permettant aux usagers de prendre rendez-vous en ligne pour différents services consulaires (passeports, visas, actes d'état civil, etc.) tout en gérant efficacement les créneaux disponibles ?

2. Besoin & Objectifs

Objectifs fonctionnels

  • Système de prise de rendez-vous en ligne
  • Gestion des différents types de prestations consulaires
  • Affichage des documents requis pour chaque service
  • Système de créneaux horaires dynamiques
  • Formulaires de demande en ligne
  • Panneau d'administration pour gérer les rendez-vous

Contraintes techniques

  • Technologies web (HTML/CSS/JS/PHP)
  • Base de données MySQL
  • Interface responsive et accessible
  • Validation des données côté client et serveur

3. Solution Proposée

Architecture de la plateforme

Application web full-stack avec :

  • Frontend : HTML5, CSS3, JavaScript (vanilla)
  • Backend : PHP pour la logique métier
  • Base de données : MySQL pour stocker les rendez-vous
  • Design : Interface moderne aux couleurs algériennes

Extrait de code - Vérification des créneaux

function checkAvailability($date, $time) {
    global $pdo;
    
    $stmt = $pdo->prepare(
        "SELECT COUNT(*) FROM appointments 
         WHERE date = ? AND time = ?"
    );
    $stmt->execute([$date, $time]);
    
    return $stmt->fetchColumn() == 0;
}

6. Difficultés & Solutions

Problème 1 : Gestion des créneaux horaires

Difficulté à éviter les réservations simultanées du même créneau.

Solution :

Implémentation d'un système de vérification en temps réel avec requêtes SQL et verrouillage des créneaux.

Problème 2 : Validation des formulaires

Sécurisation des données entrées par les utilisateurs.

Solution :

Double validation : JavaScript côté client + PHP avec requêtes préparées côté serveur.
